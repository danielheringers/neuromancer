# Backlog MVP Priorizado

## P0 (obrigatório)
1. Definir contrato de ações e decisões de policy.
2. Implementar perfis `read_only`, `read_write_with_approval`, `full_access`.
3. Bloqueio de path fora do workspace.
4. Política de rede ativa no MVP.
5. Log de auditoria JSONL append-only.
6. Fluxo de aprovação fim-a-fim na UI.
7. Adapter funcional para `codex-cli`.
8. Adapter funcional para `claude-code`.
9. Renderização terminal mínima estável.
10. Testes básicos cross-platform.

## P1 (importante)
1. Diff preview por arquivo/hunk.
2. Timeline de execução no painel lateral.
3. Mensagens de erro amigáveis para iniciantes.
4. Política por projeto (arquivo de configuração).
5. Mecanismo de cancelamento seguro de tarefa.

## P2 (evolução)
1. Perfis avançados de política por equipe.
2. Sugestões de comando assistidas por contexto.
3. Modo onboarding guiado interativo.

## Definição de pronto (DoD) para itens P0
1. Testes automáticos cobrindo fluxo principal.
2. Telemetria local e logs válidos.
3. Funcionamento demonstrável em Windows/macOS/Linux.
4. Documentação de uso atualizada.
